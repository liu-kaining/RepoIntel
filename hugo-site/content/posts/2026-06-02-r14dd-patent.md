<section class="content-panel content-panel--audit" id="audit">
<header class="panel-head">
  <span class="panel-icon" aria-hidden="true">⚙</span>
  <h2 class="panel-title">CTO 级技术审计</h2>
</header>
<div class="panel-body prose">
<p class="audit-callout audit-callout--intro">入口 <code class="code-ref">src/bin/patent/main.rs:78-133</code> 将管道分为四阶段——(1) 并发扇出源搜索 + 嵌入模型加载 <code class="code-ref">tokio::join!</code>，(2) 语义排名，(3) Ollama 或数据判定，(4) TUI/JSON 输出。lib/bin 分离清晰，可测核心全在库中，二进制仅做 CLI 胶水与 TUI 渲染。</p>
<p class="audit-callout audit-callout--highlight"><code class="code-ref">src/verdict.rs:154-170</code> 的 <code class="code-ref">floor_level</code> 函数用嵌入相似度数据兜底模型判定——当模型低估饱和度时（例如给出 Open 但有 ≥2 个 sim≥0.55 的匹配），代码强制将级别上浮至 Crowded，并替换为安全的数据衍生标题。这是一套两层防御：prompt 规则 + 代码级后置校验，且测试 <code class="code-ref">tests/verdict.rs:165-192</code> 验证了模型误判时的兜底行为。</p>
<p class="audit-callout audit-callout--highlight"><code class="code-ref">src/sources/mod.rs:36-142</code> 的 <code class="code-ref">detect_sources</code> 用词边界匹配（<code class="code-ref">idea_contains</code> 函数处理子串误匹配）和领域推断两层逻辑自动选择注册中心，同时保证 GitHub+HackerNews 始终在列；<code class="code-ref">tests/sources.rs</code> 中有专门测试确保每个 <code class="code-ref">Source</code> 变体至少被一个查询选中（防「宣传但永远不搜索」的回归）。</p>
<p class="audit-callout audit-callout--doubt"><code class="code-ref">src/sources/pypi.rs:32-43</code> 和 <code class="code-ref">src/sources/go.rs:25-28</code> 均依赖 HTML 抓取（scraper + CSS 选择器），PyPI/Go 任意改版即静默返回空结果。模块注释承认「brittle by nature」，但未见降级提示逻辑——用户可能误以为该生态无工具。建议在 <code class="code-ref">SearchOutcome.failed</code> 中增加「degraded」状态区分网络失败与解析为空。</p>
<p class="audit-callout audit-callout--doubt"><code class="code-ref">src/sources/mod.rs:151-165</code> 的 <code class="code-ref">search_sources</code> 对失败源仅重试一次且间隔固定 800ms，无指数退避或并发限流；若 11 个源同时命中（如无语言信号的宽泛查询），单次请求无超时保护（虽 <code class="code-ref">http_client</code> 设了 10s timeout），在 GitHub 未设 token 时 10 req/min 限制极易触发，此时返回的 rate limit 错误会走 Parse 错误而非优雅降级。</p>
<p>1. 将 PyPI/Go 的 HTML 抓取失败与网络错误区分——增加 <code class="code-ref">Degraded(String)</code> variant 到 <code class="code-ref">SearchOutcome</code>，TUI 中以灰色标注「解析失败」而非仅「not reached」。
2. 给所有 source adapter 加 <code class="code-ref">with_base_url</code> 测试覆盖后，考虑在 <code class="code-ref">search_sources</code> 中对 GitHub 无 token 时自动降频（如仅搜索 10 个结果而非 20）。
3. 嵌入模型加载 <code class="code-ref">fastembed::TextEmbedding::try_new</code> 在 <code class="code-ref">src/rank.rs:118-128</code> 中同步执行，首次下载 80MB 时虽有 <code class="code-ref">tokio::task::spawn_blocking</code> 包裹但仍会阻塞一个线程池线程；建议在 CLI 中提前打印进度条而非仅一行 eprintln。</p>
</div>

<header class="panel-head">
  <span class="panel-icon" aria-hidden="true">⚠</span>
  <h2 class="panel-title">隐藏风险与雷点</h2>
</header>
<div class="panel-body prose">
<ul class="risk-list"><li>PyPI/Go 依赖 HTML 抓取，注册中心改版后会静默返回空结果而非报错，用户可能误判为空间空白</li><li>项目创建仅 3 天，仅 3 个 fork，维护者集中度极高（单人项目），长期维护和社区贡献存在不确定性</li><li>未审阅到 crate 中的 demo.tape / CLAUDE.md 等非代码文件，无法评估是否有 AI 生成痕迹或外部依赖风险</li><li>summary 过长，可能含废话</li></ul>
